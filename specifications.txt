1. VISÃO GERAL
────────────────────────────────────────────────────────────────────────────────

Vitrine3D é uma plataforma SaaS para profissionais de impressão 3D criarem e
gerenciarem sua vitrine de produtos online. Cada vendedor possui uma URL própria
(ex: vitrine3d.com/printlab3d) onde exibe seu catálogo e recebe pedidos de
orçamento diretamente via WhatsApp, sem intermediários.

Tecnologias do frontend:
  - React 19 + TypeScript
  - Vite 6
  - Tailwind CSS v4
  - React Router DOM v7
  - Axios

Backend previsto (ainda não implementado):
  - Spring Boot (Java)
  - PostgreSQL
  - MinIO (armazenamento de imagens)
  - JSoup (scraper do MakerWorld)

2. CASOS DE USO ESPERADOS
────────────────────────────────────────────────────────────────────────────────

UC-01 | VISITANTE visualiza a landing page da plataforma
  - Acessa vitrine3d.com
  - Vê apresentação da plataforma com os diferenciais
  - Pode clicar em "Seja um Vendedor" para criar conta
  - Pode clicar em "Ver exemplo de vitrine" para visualizar uma loja demo

UC-02 | VISITANTE acessa a vitrine de um vendedor
  - Acessa vitrine3d.com/{slug-da-loja}
  - Visualiza perfil da loja (nome, descrição, avatar)
  - Filtra produtos por categoria
  - Vê produtos em destaque no carrossel/bento hero
  - Clica em um produto e vê o modal com todos os detalhes
  - Clica em "Solicitar Orçamento" e é redirecionado ao WhatsApp

UC-03 | VENDEDOR cria conta na plataforma
  - Acessa /admin/register
  - Preenche nome, e-mail, senha, nome da loja, slug, cidade , estado,  WhatsApp
  - Recebe confirmação e é redirecionado ao painel

UC-04 | VENDEDOR faz login no painel
  - Acessa /admin/login
  - Preenche e-mail e senha
  - Em caso de sucesso é redirecionado ao painel de produtos
  - Em caso de erro vê mensagem de feedback
  - Sessão persiste via JWT no localStorage

UC-05 | VENDEDOR gerencia seus produtos
  - Visualiza lista de todos os produtos com miniatura, nome, material e status
  - Adiciona novo produto manualmente ou via importação do MakerWorld
  - Edita informações de um produto existente
  - Ativa/desativa visibilidade de um produto na vitrine
  - Exclui um produto

UC-06 | VENDEDOR importa produto do MakerWorld
  - Cola a URL de um modelo do MakerWorld no formulário
  - O backend faz scraping e retorna nome, descrição e URL da imagem
  - O formulário é preenchido automaticamente
  - O vendedor complementa as informações (material, dimensões, categoria)
  - Salva o produto

UC-07 | VENDEDOR gerencia categorias
  - Visualiza categorias globais da plataforma
  - Cria categorias personalizadas para sua loja
  - Vincula produtos às categorias

UC-08 | VENDEDOR configura sua loja
  - Atualiza nome da loja, descrição e número de WhatsApp
  - Faz upload ou define URL do logo
  - Visualiza e altera seu slug de URL

UC-09 | SISTEMA renova sessão automaticamente
  - Sessão é recarregada do localStorage ao abrir o navegador
  - Token expirado ou inválido redireciona para login automaticamente (401)

5. BACKEND — NÃO IMPLEMENTADO
────────────────────────────────────────────────────────────────────────────────

Toda a camada de servidor ainda precisa ser construída. Os contratos de API
foram definidos no FRONTEND_CONTEXT.md e os serviços do frontend já estão
preparados para consumi-los.

Endpoints necessários (Spring Boot):

  Autenticação:
    POST /api/auth/login       → { email, password } → { token, user }
    POST /api/auth/register    → { ...UserPayload } → { token, user }

  Lojas:
    GET  /api/stores/:slug     → StoreInfo pública do vendedor
    PATCH /api/stores/:id      → Atualizar dados da loja (autenticado)

  Produtos:
    GET  /api/products         → Listar produtos do vendedor autenticado
    POST /api/products         → Criar produto
    PATCH /api/products/:id    → Editar produto
    DELETE /api/products/:id   → Excluir produto
    POST /api/products/scrape  → { url } → { name, description, imageUrl } (JSoup)

  Categorias:
    GET  /api/categories       → Listar categorias (globais + do vendedor)
    POST /api/categories       → Criar categoria personalizada
    DELETE /api/categories/:id → Excluir categoria do vendedor
